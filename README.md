# Android Intrusion Log Parser to CSV

A resilient, command-line Python utility designed to parse highly dynamic Android log datasets stored as row-by-row JSON records within `.txt` files. This script automatically crawls an entire directory, normalizes disparate network and system events into a unified data structure, and merges them into a clean, review-ready CSV spreadsheet.

## 📱 How to Retrieve Logs

The JSON data parsed by this script consists of security and connection telemetry generated by Android's Advanced Protection subsystem. You can locate and export these logs from your device by following these steps:

1. Open your device's **Settings** app.
2. Tap **Security & privacy**.
3. Tap **Advanced Protection**.
4. Tap **Intrusion Logging**.
5. Tap **Access logs** to view, manage, and download the logs onto your machine as `.zip` file containing `.txt` logs.

## ✨ Features

* **Folder Batch Processing:** Automatically scans any target directory for `.txt` files and processes them in alphabetical order.
* **Source Tracking:** Appends a `source_file` column to every single CSV record so you know exactly which log file a data point originated from.
* **Epoch Time Normalization:** Dynamically handles both millisecond (13-digit) and high-resolution nanosecond (19-digit) epoch timestamps, translating them into a unified human-readable format (`YYYY-MM-DD HH:MM:SS.mmm UTC`).
* **Resilient Semi-Dynamic Schema:** * Parses `dns_event` strings and cleans leading slashes (`/`) from IP addresses.
  * Parses `connect_event` payloads and standardizes connection sockets (`IP:Port`).
  * Extracts nested information from `security_event` blobs (e.g., `app_process_start`, `key_generated`, `key_destruction`), mapping execution processes straight into your target table.
* **Zero-Loss Catch-All:** Any unexpected log format or extra nested values are safely encoded as localized JSON objects inside an `extra_details` column instead of causing the script to crash.

## 🚀 Getting Started

### 1. Structure Your Directory
Place your downloaded text files containing the row-by-row JSON entries inside a common folder (e.g., `logs_folder`). 

Your directory structure should look something like this:
```text
parser/
├── parse_intrusion_logs.py
└── logs_folder/
    ├── 2016-06-16(1).txt
    ├── 2016-06-16(2).txt
    ├── 2016-06-16(3).txt
    ├── 2016-06-16(4).txt
    └── 2016-06-16.txt

Sample command for the above:

python parse_intrusion_logs.py -o output.csv
